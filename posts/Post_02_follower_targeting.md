## 1. 팔로워 공격 구현 중 생긴 문제

[제1편]에서 여러 마리의 팔로워를 내 뒤에 소환하는 데 성공했습니다. 이제 전투가 벌어지면 팔로워들이 적을 향해 돌진(대시 공격)해야 합니다. 
하지만 아무런 규칙 없이 "제일 가까운 적을 공격해!"라고만 명령을 내리면, **5마리의 팔로워가 오직 1마리의 적에게만 우르르 몰려가는** 비효율적인 상황이 발생합니다. 나머지 적들은 공격받지 않고 플레이어를 향해 걸어오게 되죠.

이 문제를 해결하기 위해, 한 명의 적은 딱 한 마리의 팔로워만 전담 마크하도록 똑똑하게 목표를 분배해 주는 **타겟팅 인공지능(Targeting AI)** 이 필요했습니다.

## 2. 중복 타겟팅을 막는 알고리즘 (TargetingAI)

모든 팔로워와 적의 거리를 계산하고 적절하게 분배하는 역할은 `TargetingAI` 라는 중앙 관리자 코드가 맡습니다.

```csharp
// Scripts/Objects/Characters/Followers/Calculator/TargetingAI.cs 내부 핵심 로직

// 1. 이미 누군가에게 지목당한 '목표물'들을 기록해두는 명부(HashSet)입니다.
private HashSet<Enemy> _targetHashSet = new HashSet<Enemy>();

// 2. 목표 분배 함수
private void AssignTargets()
{
    foreach (Follower follower in _activeFollowers)
    {
        // 이미 돌진 중인 팔로워는 건너뜁니다.
        if (follower.IsDashCheck()) continue;

        // 거리가 가장 가까운 순서대로 정렬된 적 목록(_targetCandidates)을 살펴봅니다.
        foreach (var candidate in _targetCandidates)
        {
            Enemy enemy = candidate.Item2;

            // 만약 이 적이 아직 아무에게도 지목당하지 않았다면! (명부에 없다면)
            if (!_targetHashSet.Contains(enemy))
            {
                follower.SetTarget(enemy);       // "너는 이 적을 공격해라!"
                _targetHashSet.Add(enemy);       // "이 적은 지목완료. 명부에 추가!"
                
                follower.CallDashAttack();       // 돌진 공격 실시!
                break; // 이 팔로워는 목표를 찾았으니 다음 팔로워로 넘어갑니다.
            }
        }
    }
}
```

이 알고리즘의 핵심은 `_targetHashSet`이라는 명부입니다. 
가장 가까운 적이라도 이미 다른 팔로워가 공격하러 가기로 예약되어 있다면, 명부에 등록되어 있기 때문에 다음 팔로워는 그 적을 깔끔하게 무시하고 두 번째로 가까운 적을 찾아 돌진하게 됩니다. 

이 코드를 적용하자 팔로워들이 각자 1명씩 적을 맡아 깔끔하게 흩어져서 공격하는 멋진 장면이 연출되었습니다.

## 3. 공격 후 제자리로 돌아오기

적을 향해 무섭게 돌진(대시 공격)한 팔로워는, 공격이 끝난 후 다시 플레이어의 등 뒤(원래 할당받았던 빈자리)로 돌아와야 합니다. 그래야 다음 전투를 준비할 수 있으니까요.

팔로워는 돌진하기 전에 **자신이 서 있던 원래 자리(Origin Position)** 를 기억해 둡니다. 공격 판정이 끝나고 적과 부딪히면, 곧바로 기억해 둔 원래 자리 방향으로 이동 방향을 휙 틀어서 플레이어에게 복귀하도록 상태(State)가 전환됩니다.
