# [제4편] 게임에 생동감 불어넣기: Cinemachine과 DOTween

> **시리즈 목차**
> - [제1편] 적을 아군으로: 포획과 팔로워 소환 파이프라인

## 1. 밋밋한 화면에 생동감을 더하는 방법

게임의 핵심 로직과 인벤토리 시스템이 훌륭하게 작동하더라도, 눈으로 보이는 연출이 심심하면 캐주얼 액션 게임 특유의 찰진 타격감과 속도감을 느낄 수 없습니다. 

모든 애니메이션과 카메라 이동을 직접 수학 코드로 계산할 수도 있지만, 개발 효율을 극대화하기 위해 **시네머신(Cinemachine)**과 **두트윈(DOTween)**을 사용했습니다.

## 2. 시네머신(Cinemachine)을 이용한 카메라 워킹

플랫포머 게임에서 플레이어를 따라다니는 카메라는 매우 중요합니다. 플레이어가 점프할 때마다 카메라가 덜컹거리고, 맵 끝으로 갔는데도 카메라가 맵 바깥의 까만 허공을 비추면 몰입감이 크게 떨어집니다.

**시네머신(Cinemachine)**은 이런 문제를 단번에 해결해 주는 유니티의 가상 카메라 시스템입니다.
- **부드러운 추적**: 코드 한 줄 없이 플레이어를 타겟(Follow Target)으로 지정하기만 하면, 캐릭터의 움직임에 맞춰 부드럽게(Dead Zone, Damping 설정) 카메라가 따라갑니다.
- **화면 흔들림 (Camera Shake)**: 보스의 거대한 공격이 바닥에 내리꽂힐 때 화면을 강하게 진동시켜 타격감을 크게 올리는 연출을 쉽게 적용할 수 있습니다.
- **영역 제한 (Confiner)**: 카메라가 절대 넘어가지 말아야 할 맵의 경계선을 지정해, 허공이 비치는 것을 방지했습니다.

## 3. DOTween을 활용한 코드 기반 애니메이션

유니티의 기본 애니메이터(Animator)는 캐릭터가 걷거나 뛰는 동작에는 훌륭하지만, "UI 창이 튀어나오기", "팔로워가 적을 향해 일직선으로 돌진하기" 같은 행동을 만들 때와 같이 섬세한 연출이 필요할 때 DOTWeen을 사용하는 게 도움이 됐습니다.

이때 **DOTween**을 사용하면 수학적인 보간(Lerp) 코드를 일일이 작성할 필요 없이, 단 몇 줄의 코드로 매우 부드럽고 다이나믹한 움직임을 만들 수 있습니다.

[제2편]에서 설명했던 팔로워의 돌진(대시) 후 복귀 로직을 DOTween으로 어떻게 구현했는지 살펴보겠습니다.

```csharp
// Scripts/Objects/Characters/Followers/Controller/FollowerController.cs 내부 대시 로직

// DOTween의 Sequence(연속된 애니메이션) 기능을 사용합니다.
Sequence seq = DOTween.Sequence();

// 1. 적(targetPos)을 향해 dashDuration 시간 동안 이동합니다.
seq.Append(transform.DOMove(targetPos, dashDuration))
   
   // 2. 적에게 닿은 후 0.5초 동안 제자리에서 대기합니다. (타격감)
   .AppendInterval(0.5f)
   
   // 3. 다시 플레이어 뒤의 원래 자리(followPos)로 이동합니다.
   .Append(transform.DOMove(followPos, dashDuration))
   
   // 4. 모든 애니메이션이 끝나면 실행할 마무리 작업을 묶어줍니다.
   .OnComplete(() =>
   {
       currentTarget = null;
       isDashing = false; // 돌진 상태 해제
   })
   .Play(); // 애니메이션 실행!
```

이전 같았으면 `Update()` 함수 안에서 남은 시간과 거리를 계산하며 수십 줄에 걸쳐 짜야 했을 돌진 로직이, DOTween 체이닝 기법 덕분에 마치 책을 읽듯이 쉽게 읽히고 유지보수도 매우 편해졌습니다.
