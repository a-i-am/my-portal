## 1. 2D 캐릭터에 관절을 달아주다 (Spine 2D)

사용한 에셋에서 스파인(Spine 2D)** 을 사용해야 했습니다. 스파인은 2D 그림에 사람처럼 뼈대(Bone)와 관절을 심어서 애니메이션을 만들어내는 도구입니다. 한 장의 그림만 있어도 뼈대의 각도만 틀어주면 부드럽게 움직이는 관절 인형 같은 캐릭터를 만들 수 있고, 게임의 흐름에 맞춰 동작의 속도를 코드로 마음대로 조절할 수 있습니다.

## 2. 유니티 애니메이터와 스파인의 딜레마

문제는 스파인으로 만든 애니메이션을 유니티(Unity)로 가져왔을 때 발생합니다.
유니티에는 캐릭터의 상태(대기 ➔ 걷기 ➔ 공격)를 화살표로 연결해 시각적으로 쉽게 관리할 수 있는 강력한 기능인 **애니메이터(Animator)** 가 있습니다. 하지만 스파인에서 가져온 데이터는 유니티 기본 애니메이터와 완벽하게 100% 호환되지는 않습니다. 스파인만의 전용 코드를 텍스트로 일일이 쳐서 애니메이션을 재생해야 하는 불편함이 생깁니다.

## 3. 두 시스템을 연결하는 다리 (SpineAnimatorBehaviour)

이 문제를 해결하기 위해 **'SpineAnimatorBehaviour'** 라는 다리 역할을 하는 코드를 직접 작성했습니다. 

유니티 애니메이터에는 각 상태(대기, 걷기 등)가 시작될 때마다 자동으로 실행되는 기능(`StateMachineBehaviour`)이 숨어 있습니다. 이 기능에 우리가 만든 다리 코드를 붙여주면 딜레마가 해결됩니다.

```csharp
// Animations/SpineAnimatorBehaviour.cs의 핵심 원리

// 1. 유니티 애니메이터에서 '공격(Attack)' 상태에 진입하면 이 코드가 자동으로 실행됩니다.
override public void OnStateEnter(Animator animator, AnimatorStateInfo stateInfo, int layerIndex)
{
    // 2. 캐릭터의 무기 정보를 확인해서 공격 속도를 계산합니다.
    float timeScale = 무기_공격_속도;
    
    // 3. 스파인 시스템에게 "지금부터 '공격' 뼈대 움직임을 재생해!" 라고 명령을 내립니다.
    spineAnimationState.SetAnimation(0, "MeleeAttack", false);
    
    // 4. 스파인 애니메이션의 재생 속도를 무기 공격 속도에 맞춰 빠르거나 느리게 조절합니다.
    trackEntry.TimeScale = timeScale;
}
```

이 방식의 장점은 다음과 같습니다.
1. **눈으로 확인하는 흐름**: 복잡한 코드 대신 유니티 애니메이터 창에서 화살표를 이어가며 "대기하다가 마우스를 누르면 공격한다"는 흐름을 시각적으로 짤 수 있습니다.
2. **부드러운 모션과 동적 속도 제어**: 실제 애니메이션 재생은 스파인 시스템이 담당하므로, 캐릭터가 무거운 칼을 들었을 때는 재생 속도를 느리게(`timeScale` 조절) 하고 가벼운 단검을 들었을 때는 재생 속도를 빠르게 코드로 조절할 수 있게 되었습니다.
