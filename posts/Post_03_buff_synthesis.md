## 1. 실시간 버프 합성(Merge) 구현

턴제 전투에서 패링(Parry)이나 회피(Dodge)에 연속으로 성공하면 "공격력 1단계 증가" 버프를 여러 개 얻을 수 있습니다. 하지만 똑같은 1단계 버프만 있으면 전략적 쾌감이 떨어집니다.

유명한 게임인 '오토체스' 류에서 1성 유닛 2개가 모이면 2성 유닛이 되듯, **'Uni Birth'** 프로젝트에서는 **동일한 등급의 버프가 2개 모이면 상위 등급의 버프 1개로 진화(합성)하는 시스템**을 고안했습니다.

- 1단계(C등급) 버프 + 1단계(C등급) 버프 = **2단계(B등급) 버프 1개**
- 2단계(B등급) 버프 + 2단계(B등급) 버프 = **3단계(A등급) 버프 1개** (MAX)

## 2. 재귀 함수(Recursion)를 활용한 연쇄 합성 로직

버프 합성은 생각보다 까다롭습니다. C등급이 들어와서 기존 C등급과 합쳐져 B등급이 되었는데, 마침 버프 창에 기존 B등급 버프가 이미 하나 있었다면? 방금 만들어진 B등급이 다시 기존 B등급과 합쳐져 곧바로 A등급이 되어야 하는 **연쇄 반응(Chain Reaction)** 이 일어납니다.

이러한 꼬리 물기 식의 합성을 가장 깔끔하게 해결할 수 있는 방법은 바로 **재귀 함수(Recursion)** 였습니다.

```cpp
// Source/UniBirth/Component/UBComboBuffComponent.cpp

FActiveBuff UUBComboBuffComponent::GetFinalMergedBuff(EComboType Type, EComboLevel Level)
{
    // 1. 현재 들어온 버프와 같은 등급(Level)의 버프가 이미 존재하는지 찾습니다.
    int32 ExistIndex = FindMergeTargetIndex(Type, Level);

    // 2. 만약 같은 등급의 버프가 있고, 아직 최고 등급(MAX)이 아니라면 합성 시작!
    if (ExistIndex != -1 && Level < EComboLevel::MAX)
    {
        TArray<FActiveBuff>& ActiveBuffs = allActiveBuffs[Type];
        
        // 기존에 있던 버프를 제거(소모)합니다.
        RemoveActiveBuff(ActiveBuffs, ExistIndex);

        // ★ 핵심: 등급을 한 단계 높여서 자기 자신(함수)을 다시 호출합니다!
        // 여기서 반환된 상위 등급 버프가 또 합성될 수 있는지 재귀적으로 검사하게 됩니다.
        return GetFinalMergedBuff(Type, (EComboLevel)((uint8)Level + 1));
    }

    // 3. 더 이상 합쳐질 같은 등급의 버프가 없다면, 최종적으로 확정된 이 버프를 반환합니다.
    return CreateActiveBuff(Type, Level);
}
```

이 재귀 로직 단 몇 줄 덕분에, C등급 버프가 여러 개 쏟아져 들어오더라도 코드가 스스로 "합성 ➔ 등급업 ➔ 또 합성 ➔ 등급업"을 연쇄적으로 처리하여 최종 버프 하나만 깔끔하게 뱉어내게 됩니다. 반복문(`while`)을 복잡하게 돌릴 때 발생할 수 있는 '무한 루프'나 '배열 인덱스 오류'의 위험도 크게 줄일 수 있었습니다.

## 3. 적용과 마무리

새롭게 탄생한 A등급, B등급 버프는 이전 글에서 다루었던 `UBBuffComponent`로 전달되어 델리게이트를 통해 실시간으로 UI 아이콘에 금빛 띠를 두르며 화려하게 갱신됩니다. 플레이어는 QTE 연속 성공이라는 컨트롤의 쾌감에 더해, 버프가 상위 단계로 쑥쑥 합성되는 전략적 시각 효과까지 동시에 즐길 수 있게 되었습니다.

이것으로 총 3편에 걸친 **Uni Birth** 프로젝트의 전투/버프 시스템 개발기를 마칩니다. 언리얼 엔진 5의 C++ 환경에서 데이터 테이블(CSV), 델리게이트, 그리고 재귀 알고리즘을 엮어 하나의 견고한 시스템을 만들어낸 값진 경험이었습니다.
