## 1. 3D 세상에 납작한 2D 종이 인형 세우기

'Tell The Story'는 화려한 3D 배경 속에서 도트 그래픽 형태의 2D 캐릭터들이 돌아다니는 아트를 가지고 있습니다. 마치 3D 공간에 납작한 종이 인형을 세워둔 것 같은 이 느낌을 구현하기 위해 **빌보드(Billboard) 기법**을 사용했습니다.

## 2. 항상 나를 바라보는 캐릭터 (SpriteBillboard)

캐릭터가 언제나 카메라(플레이어의 시선)를 쳐다 보도록 각도를 계속 틀어주어야 합니다.

이를 위해 `SpriteBillboard` 스크립트를 작성하여 캐릭터에 부착했습니다.

```csharp
// Scripts/Lobby/Player/SpriteBillboard.cs

public class SpriteBillboard : MonoBehaviour
{
    [SerializeField] bool freezeXZAxis = true;

    void Update()
    {
        if (freezeXZAxis)
        {
            // 캐릭터가 땅에 똑바로 서 있도록 X, Z축은 고정하고, 
            // 좌우(Y축) 회전만 카메라의 회전값과 똑같이 맞춥니다.
            transform.rotation = Quaternion.Euler(0f, Camera.main.transform.rotation.eulerAngles.y, 0f);
        }
        else
        {
            // 하늘을 날거나 완전히 누워야 하는 경우 카메라의 모든 회전값을 따라갑니다.
            transform.rotation = Camera.main.transform.rotation;
        }
    }
}
```

이 코드가 켜져 있는 한, 카메라가 빙글빙글 돌아도 캐릭터 그림은 항상 카메라를 정면으로 쳐다보게 됩니다.

## 3. 뒤통수도 보여줘야 한다: 8방향 스프라이트 제어

빌보드 기법으로 캐릭터가 항상 정면으로 보이게 만들었지만, 새로운 문제가 생겼습니다. 캐릭터가 우리를 등지고 앞으로 걸어가도, 그림은 계속 앞모습(정면)만 보인다는 점입니다.

이를 해결하기 위해, 캐릭터가 바라보는 실제 3D 방향과 카메라가 쳐다보는 방향의 **각도 차이**를 계산해야 합니다. 저희는 `EightDirectionalSpriteSystem`이라는 외부 에셋과 구조를 활용해 이 계산을 처리했습니다.

1. **8장의 그림 준비**: 앞, 뒤, 좌, 우, 그리고 대각선 4방향까지 총 8가지 방향의 캐릭터 그림을 준비합니다.
2. **각도 계산**: "현재 카메라가 캐릭터의 어디에 있는가?"를 수학적으로 계산합니다. 만약 카메라가 캐릭터의 등 뒤에 있다면 180도, 오른쪽에 있다면 90도의 차이가 납니다.
3. **그림 교체**: 계산된 각도에 따라 360도를 8등분하여, 카메라가 캐릭터의 뒤통수 쪽에 있다면 '뒷모습' 그림을, 대각선에 있다면 '대각선' 그림을 화면에 보여주도록 실시간으로 이미지를 갈아 끼웁니다.

이 로직 덕분에, 물리적인 3D 모델링 없이 납작한 2D 이미지 8장만으로 3D 공간에서 완벽하게 자연스러운 캐릭터 이동을 연출할 수 있었습니다.
